# `.mdreview` Sidecar Spec

Spec version: 0.1

Margent stores Markdown documents as ordinary files and review state as JSON sidecars under a sibling `.mdreview/` directory at the workspace root. The Markdown file remains the source text. Sidecars are the source of truth for comments, proposal state, agent cursors, review memory, and provenance metadata.

## Layout

- `.mdreview/documents/*.json`: indexed Markdown document records.
- `.mdreview/threads/*.json`: anchored comment threads and messages.
- `.mdreview/proposals/*.json`: pending, accepted, rejected, stale, or failed revision proposals.
- `.mdreview/events.ndjson`: append-only workspace event log, one event object per line.
- `.mdreview/agents/*.json`: per-agent cursor records over the event log.
- `.mdreview/authorship/*.json`: accepted proposal spans with provider attribution.
- `.mdreview/snapshots/*.json`: pre-mutation document snapshots used by `margent revert-last`.
- `.mdreview/manual.md`: optional standing review instructions.
- `.mdreview/memory.md`: review memory generated by `margent codify`, safe for humans to edit.
- `.mdreview/skills/*.md`: selectable review passes for provider actions.

## Versioning

Each record with a `schemaVersion` field declares the version of that record shape. Current versions are:

- Document: `1`
- Thread: `6`
- Proposal: `1`
- Authorship map: `1`

Older thread records are normalized lazily by the CLI and Tauri app when touched. Consumers should preserve unknown fields they do not understand and should prefer stable IDs over display ordinals.

## Anchors

Thread and authorship anchors store UTF-16 offsets, line/column hints, quote/context windows, heading path, a block fingerprint, base content hash, and best-effort reattachment state. `anchor.kind` is currently `text_span`, `document`, `footnote_reference`, `footnote_definition`, or `authorship_span`; consumers must tolerate future kind strings.

## Events

`events.ndjson` is append-only. Agent cursors store the last seen event id and allow incremental sync. Event types are stringly typed by design; known examples include `thread.created`, `thread.replied`, `thread.resolved`, `thread.reopened`, `thread.deleted`, `proposal.accepted`, `proposal.rejected`, and `proposal.stale`.
